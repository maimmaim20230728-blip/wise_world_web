# Wise World 🦉

世界中の、学校に通えない子どもたちが「幸せに・健康に生きるための基礎知識」を学ぶための、**軽量・完全オフライン**のクイズアプリです。

- 4択クイズ／ランダム出題（重複なし）
- EXP（経験値）とレベルアップ
- 解説をじっくり読ませる仕掛け（3秒ロック＋最後までスクロール）
- 広告なし・課金なし・通信なし
- 1コードベースで **Web版** と **Google Play版** の両方に展開できます

---

## 1. これは何でできているか（設計の判断）

計画書では Flutter を想定していましたが、本プロトタイプは **素のHTML/CSS/JavaScript（PWA）** で作っています。理由：

| | このHTML/PWA方式 | Flutter Web |
|---|---|---|
| Web初回読み込み | 約 30KB・一瞬 | 約 2MB以上・重い |
| 低スペック端末・弱い回線 | ◎ | △ |
| オフライン | ◎（標準で動く） | ○ |
| Web版とPlay版の共通化 | ◎（同じコード） | ◎ |

「世界向け・軽量」という目的には、軽いHTMLのほうが適しています。4択クイズは Flutter のような重いネイティブUIを必要としません。
（フルネイティブが必要になったときのために Flutter 版も後から作れます。）

## 3. 動かしかた（Web版）

### いちばん簡単：ファイルを開くだけ
`index.html` をダブルクリックしてブラウザで開けば、そのまま動きます（通信不要）。

### おすすめ：ローカルサーバーで開く（PWA／オフライン機能を確認するため）
Service Worker は `http(s)` 配信時のみ有効になります。

```powershell
cd C:\Users\puipu\wise_world
npx serve .
# 表示された http://localhost:3000 などをブラウザで開く
```

### 公開（無料でできます）
`wise_world` フォルダごと、以下のどれかにアップロードするだけ：
- Netlify / Cloudflare Pages / GitHub Pages など静的ホスティング
- HTTPSであれば、スマホで「ホーム画面に追加」するとアプリのように起動します

## 4. Google Play版の作りかた（同じコードを使う）

オフライン重視なので **Capacitor** を推奨します（アプリ内に一式を同梱＝通信ゼロで動く、本物の `.aab` が作れる）。

```powershell
# 1) Capacitor を用意
cd C:\Users\puipu\wise_world
npm init -y
npm install @capacitor/core @capacitor/cli @capacitor/android
npx cap init "Wise World" com.soyogi.wiseworld --web-dir=.

# 2) Androidプロジェクトを生成して、いまのWeb資産を取り込む
npx cap add android
npx cap copy

# 3) Android Studio で開いて署名つきの .aab をビルド
npx cap open android
#   Build > Generate Signed Bundle / APK > Android App Bundle
```

> Playストア用には、512×512 などの **PNGアイコン** が別途必要です（今は軽量なSVGのみ）。
> 必要になったら PNG 書き出しスクリプトを追加します。声をかけてください。

別案：完全ネイティブにしたい場合は、計画書どおり **Flutter** で `main.dart` を作る道もあります（`C:\flutter` に導入済み）。

## 5. 問題の追加・修正・翻訳

各言語の `questions.<code>.js` を編集します（日本語は `questions.ja.js`）。1問の形：

```js
{
  id: 21,
  topic: "あんぜんな みず",          // 小見出し
  emoji: "💧",                       // イラスト代わりの絵文字（全言語で共通）
  question: "問題文…",
  choices: ["え1","え2","え3","え4"], // ちょうど4つ
  answerIndex: 0,                    // 正解の番号 (0〜3／全言語で共通)
  explanation: "丁寧な長文の解説…"
}
```
> `id`・`answerIndex`・`emoji` は全言語で同じにしてください（それ以外を訳します）。

## 5.5 多言語化（世界向けの核）

12の主要言語に対応予定で、**現在は 日本語・英語** が入っています（残りは順次追加）。

- **画面の文言** … [i18n.js](i18n.js)（全言語のUI文字列）
- **問題データ** … `questions.<code>.js`（言語ごと。選んだ言語のぶんだけ読み込む＝軽い）
- 右上の 🌐 ボタンで言語切替。初回は端末の言語を自動判定し、無ければ英語
- アラビア語など右→左の言語にも対応（`dir:"rtl"`）

### 言語をひとつ足す手順（3ステップ）
1. `i18n.js` の `langs` に `{ code, native, dir }` を1行足す
2. `i18n.js` の `ui` に、その言語のUI文字列ブロックを足す（`en` をコピーして訳す）
3. `questions.<code>.js` を作る（`questions.en.js` をコピーして訳す）
4. `sw.js` の `ASSETS` にそのファイルを足し、`CACHE` の版を上げる
> 自動で言語ボタンに出ます。アプリ本体のコードは触りません。

### ⚠ 翻訳の正確さについて（重要）
解説は **命に関わる健康・安全** の内容です。日本語以外は **AIによる翻訳** で、
各言語の画面に「母語話者・専門家の確認を推奨」と明記しています。
**公開・配布の前に、各言語を母語話者や保健の専門家に確認してもらってください。**

## 6. EXP・レベルのルール（学びを正しく評価する設計）

- 正解：**+10 EXP** ／ 不正解でも：**+7 EXP**（差は小さく。挑戦と学びを尊重）
- ★**EXPは解答した瞬間には入りません。** 解説を最後まで読んで「つぎへ」を押した
  ときに、**次の問題へ移る間に**EXPが増え、レベルが上がります。
  → 「答えたこと」ではなく「読んで学んだこと」に報酬を与えるためです。
- **100 EXP** ごとにレベルアップ（超過分は次のレベルへ繰り越し）
- 進捗は端末内（localStorage）に自動保存

## 6.5 サウンド

- すべて **Web Audio API による合成**（音声ファイル不要・容量ゼロ・著作権の心配なし・完全オフライン）
- タイトル：明るい約20秒の長めループ。クイズ・解説中：静かなループ
- 効果音：タップ／正解／不正解／EXP獲得／レベルアップ
- 右上の 🔊 ボタンでミュート切替（設定は保存）。ブラウザの自動再生制限のため、
  **最初のタップで音が鳴り始めます**
- 実装は [audio.js](audio.js)。曲・効果音はここで調整できます
